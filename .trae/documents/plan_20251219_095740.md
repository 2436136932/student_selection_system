# 学生成绩权限控制实现计划

## 一、问题分析

### 核心问题
当前成绩管理系统存在权限控制漏洞，学生可以查看所有学生的成绩，主要原因：

1. **关键接口缺少权限限制**：`/api/scores/page` 接口无 `PreAuthorize` 注解，任何人可访问
2. **学生身份未用于数据过滤**：即使学生登录，系统也未根据其身份过滤数据
3. **部分接口权限设计不合理**：如 `student/{studentId}` 接口的权限表达式可能存在逻辑问题

### 具体问题接口

| 接口 | 问题 |
|------|------|
| `/api/scores/page` | 无权限限制，可获取所有成绩 |
| `/api/scores/course/{courseId}` | 无权限限制，可获取课程所有学生成绩 |
| `/api/scores/student/{studentId}` | 权限表达式可能不正确 |
| `/api/scores/statistics` | 无数据过滤，学生可查看所有统计数据 |

## 二、解决方案

### 核心思路

1. **身份识别与权限控制**：利用 Spring Security 的认证信息获取当前用户身份
2. **数据过滤机制**：
   - ADMIN/TEACHER：可查看所有成绩
   - STUDENT：仅能查看自己的成绩
3. **统一的学生身份获取**：通过用户名（学号）关联学生信息

### 具体实现方案

#### 1. 修改 `ScoreController` 权限控制

**1.1 添加依赖注入**
```java
@Autowired
private UserService userService;

@Autowired
private AuthenticationManager authenticationManager;
```

**1.2 新增学生身份获取方法**
```java
/**
 * 获取当前学生的学号（仅适用于学生角色）
 */
private String getCurrentStudentNumber() {
    Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
    if (authentication != null && authentication.getAuthorities().stream().anyMatch(a -> a.getAuthority().equals("ROLE_STUDENT"))) {
        return authentication.getName(); // 假设学生用户名为学号
    }
    return null;
}
```

**1.3 修改 `/api/scores/page` 接口**
```java
@PreAuthorize("hasRole('ADMIN') or hasRole('TEACHER') or hasRole('STUDENT')")
@GetMapping("/page")
public ResponseEntity<IPage<Score>> findScoresByPage(
        @RequestParam(defaultValue = "1") Integer current,
        @RequestParam(defaultValue = "10") Integer size,
        @RequestParam(required = false) String studentNumber,
        @RequestParam(required = false) String courseCode,
        @RequestParam(required = false) String semester) {
    try {
        // 如果是学生角色，强制使用当前学生的学号
        String currentStudentNumber = getCurrentStudentNumber();
        if (currentStudentNumber != null) {
            studentNumber = currentStudentNumber;
        }
        IPage<Score> scores = scoreService.findScoresByPage(current, size, studentNumber, courseCode, semester);
        return new ResponseEntity<>(scores, HttpStatus.OK);
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

**1.4 修改 `/api/scores/course/{courseId}` 接口**
```java
@PreAuthorize("hasRole('ADMIN') or hasRole('TEACHER') or hasRole('STUDENT')")
@GetMapping("/course/{courseId}")
public ResponseEntity<List<Score>> findScoresByCourseId(@PathVariable("courseId") Long courseId) {
    try {
        List<Score> scores = scoreService.findScoresByCourseId(courseId);
        
        // 如果是学生角色，过滤当前学生的成绩
        String currentStudentNumber = getCurrentStudentNumber();
        if (currentStudentNumber != null) {
            scores = scores.stream()
                .filter(score -> currentStudentNumber.equals(score.getStudentNumber()))
                .collect(Collectors.toList());
        }
        
        if (scores.isEmpty()) {
            return new ResponseEntity<>(HttpStatus.NO_CONTENT);
        }
        return new ResponseEntity<>(scores, HttpStatus.OK);
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

**1.5 修复 `/api/scores/student/{studentId}` 接口**
```java
@PreAuthorize("hasRole('ADMIN') or hasRole('TEACHER') or hasRole('STUDENT')")
@GetMapping("/student/{studentId}")
public ResponseEntity<List<Score>> findScoresByStudentId(@PathVariable("studentId") Long studentId) {
    try {
        // 如果是学生角色，验证是否访问自己的成绩
        String currentStudentNumber = getCurrentStudentNumber();
        if (currentStudentNumber != null) {
            // 获取当前学生的ID
            Student currentStudent = studentService.findStudentByStudentNumber(currentStudentNumber)
                .orElseThrow(() -> new RuntimeException("Student not found"));
            if (!currentStudent.getId().equals(studentId)) {
                return new ResponseEntity<>(HttpStatus.FORBIDDEN);
            }
        }
        
        List<Score> scores = scoreService.findScoresByStudentId(studentId);
        if (scores.isEmpty()) {
            return new ResponseEntity<>(HttpStatus.NO_CONTENT);
        }
        return new ResponseEntity<>(scores, HttpStatus.OK);
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

**1.6 修改 `/api/scores/statistics` 接口**
```java
@PreAuthorize("hasRole('ADMIN') or hasRole('TEACHER') or hasRole('STUDENT')")
@GetMapping("/statistics")
public ResponseEntity<java.util.Map<String, Object>> getScoreStatistics(
        @RequestParam(required = false) String studentNumber,
        @RequestParam(required = false) String courseCode,
        @RequestParam(required = false) String semester) {
    try {
        // 如果是学生角色，强制使用当前学生的学号
        String currentStudentNumber = getCurrentStudentNumber();
        if (currentStudentNumber != null) {
            studentNumber = currentStudentNumber;
        }
        java.util.Map<String, Object> statistics = scoreService.getScoreStatistics(studentNumber, courseCode, semester);
        return ResponseEntity.ok(statistics);
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

#### 2. 其他接口处理

对类似的接口如 `/api/scores/student/{studentId}/course/{courseId}`、`/api/scores/student/{studentId}/semester/{semester}` 等，采用相同的处理方式：
- 添加 `PreAuthorize` 注解
- 如果是学生角色，验证并过滤数据

## 三、实现效果

1. **ADMIN/TEACHER**：可查看所有成绩和统计数据
2. **STUDENT**：
   - 只能查看自己的成绩
   - 调用分页接口时自动过滤为自己的成绩
   - 无法查看其他学生的成绩详情
   - 只能查看自己的成绩统计

## 四、验证方法

1. 使用学生账号登录系统
2. 访问成绩管理页面
3. 检查是否只能看到自己的成绩
4. 尝试直接调用API（如使用Postman）验证权限控制
5. 使用管理员账号验证仍能查看所有成绩

## 五、代码优化建议

1. **统一权限处理**：考虑使用 AOP 或过滤器统一处理权限逻辑
2. **缓存当前学生信息**：避免重复查询数据库获取学生信息
3. **完善日志记录**：添加权限验证日志，便于调试和审计
4. **前端配合优化**：前端也可添加逻辑，隐藏不必要的操作按钮

## 六、预期收益

1. **增强数据安全性**：保护学生隐私，符合数据保护法规
2. **提升系统可信度**：让用户更信任系统，愿意使用
3. **规范权限管理**：建立统一的权限控制机制
4. **便于后续扩展**：为其他功能的权限控制提供参考

通过以上修改，可以确保学生在成绩管理中只能看到自己的成绩，同时不影响管理员和教师的正常使用。

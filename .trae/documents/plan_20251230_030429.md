### 问题分析

学生和教师修改邮箱和手机号时出现403 Forbidden错误，原因是UserController.java中的PUT /api/users/{id}端点权限控制过于严格，只允许管理员访问：

```java
@PutMapping("/{id}")
@PreAuthorize("hasRole('ADMIN')")
public ResponseEntity<User> updateUser(@PathVariable Long id, @RequestBody User user) {
    // ...
}
```

### 解决方案

修改PUT /api/users/{id}端点的权限控制，允许：

1. 管理员可以修改任何用户的信息
2. 普通用户（学生、教师）只能修改自己的信息

### 代码修改

**文件**：`src/main/java/com/example/studentselectionsystem/controller/UserController.java`

**位置**：第41-49行

**修改内容**：

* 更新@PreAuthorize注解，添加条件判断

* 允许用户修改自己的信息（通过比较当前登录用户名和要修改的用户的用户名）

### 修复后代码

```java
/**
 * 更新用户信息
 */
@PutMapping("/{id}")
@PreAuthorize("hasRole('ADMIN') or (authentication.principal.username == @userService.findUserById(#id).get().getUsername())")
public ResponseEntity<User> updateUser(@PathVariable Long id, @RequestBody User user) {
    User updatedUser = userService.updateUser(id, user);
    if (updatedUser != null) {
        return new ResponseEntity<>(updatedUser, HttpStatus.OK);
    }
    return new ResponseEntity<>(HttpStatus.NOT_FOUND);
}
```

### 预期效果

1. **管理员**：可以修改任何用户的信息
2. **教师**：只能修改自己的信息
3. **学生**：只能修改自己的信息
4. 修改邮箱和手机号时不会再出现403 Forbidden错误

### 验证方法

1. 以学生身份登录系统
2. 进入个人中心，修改邮箱或手机号
3. 点击"保存修改"按钮
4. 确认修改成功，没有出现403错误
5. 以教师身份重复上述步骤

# 实现聊天中心未读消息显示功能

## 问题分析
当前聊天中心已经实现了基本的聊天功能，但缺少未读消息提醒功能。用户需要知道是否有未读消息，以及未读消息的数量，以便及时查看。

## 解决方案
1. **在App.vue中添加未读消息统计和显示**
   - 在聊天图标上显示未读消息小红点和数量
   - 实现未读消息数量的实时更新

2. **实现未读消息统计逻辑**
   - 新增获取未读消息数量的API调用
   - 监听WebSocket消息，实时更新未读消息数量
   - 当用户打开聊天窗口时，标记消息为已读

3. **更新聊天相关组件**
   - 在ChatView.vue中添加未读消息处理逻辑
   - 在ChatWindow.vue中实现消息已读标记功能
   - 更新会话列表，显示每个会话的未读消息数量

## 具体修改
1. **修改App.vue**
   - 添加未读消息数量的响应式变量
   - 在聊天图标上显示小红点和数量
   - 实现获取未读消息数量的方法
   - 监听路由变化，更新未读消息数量

2. **修改ChatView.vue**
   - 添加获取未读消息数量的方法
   - 当选择会话时，标记该会话的消息为已读
   - 监听WebSocket消息，更新未读消息数量

3. **修改ChatWindow.vue**
   - 添加标记消息为已读的方法
   - 当用户进入聊天窗口时，标记历史消息为已读

4. **修改ChatList.vue**
   - 在会话列表中显示每个会话的未读消息数量

## 预期效果
- 当有未读消息时，聊天图标上会显示小红点和未读消息数量
- 点击聊天图标进入聊天中心后，未读消息数量会更新
- 进入具体聊天窗口后，该会话的未读消息会被标记为已读
- 会话列表中会显示每个会话的未读消息数量

## 代码修改示例
```javascript
// 在App.vue中添加未读消息相关代码
const unreadCount = ref(0)

// 获取未读消息数量
const getUnreadMessageCount = async () => {
  try {
    const response = await fetch('http://localhost:8080/api/chats/unread-count', {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    })
    if (response.ok) {
      const data = await response.json()
      unreadCount.value = data.unreadCount
    }
  } catch (error) {
    console.error('获取未读消息数量失败:', error)
  }
}

// 在聊天图标上显示未读消息数量
<el-dropdown-item command="chat">
  <el-icon><ChatLineRound /></el-icon>
  聊天中心
  <el-badge v-if="unreadCount > 0" :value="unreadCount" class="unread-badge" />
</el-dropdown-item>
```

# 奖项结束自动发送邮件通知功能实现

## 一、功能需求分析

当一个奖项到达结束时间后，系统需要：
1. 自动检测奖项结束时间
2. 确定获奖学生名单
3. 向获奖学生发送邮件通知

## 二、实现方案

### 1. 技术栈和依赖

**现有技术栈**：
- Spring Boot 3.1.5
- MyBatis Plus ORM
- MySQL 数据库

**新增依赖**：
- Spring Boot Starter Mail（用于邮件发送）
- 定时任务支持（Spring 内置）

### 2. 数据库设计

**现有相关表**：
- `award`（奖项表）：包含奖项信息，需确认是否有结束时间字段
- `student_award_records`（获奖记录表）：存储获奖学生信息
- `student`（学生表）：包含学生基本信息，需确认是否有邮箱字段

### 3. 实现步骤

#### 3.1 检查现有实体类

**检查 `Award` 实体类**：
- 确认是否包含 `endTime` 字段
- 如果没有，需要添加该字段

**检查 `Student` 实体类**：
- 确认是否包含 `email` 字段
- 如果没有，需要添加该字段

#### 3.2 配置邮件发送功能

**添加依赖**：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-mail</artifactId>
</dependency>
```

**配置邮件属性**：
在 `application.yml` 或 `application.properties` 中添加邮件发送配置

#### 3.3 创建邮件发送服务

**创建 `EmailService`**：
- 实现邮件发送方法
- 支持邮件模板

#### 3.4 实现定时任务

**创建 `AwardScheduler` 类**：
- 使用 `@Scheduled` 注解实现定期检查
- 检查所有奖项的结束时间
- 对于到达结束时间的奖项，获取获奖学生列表
- 调用邮件发送服务发送通知

#### 3.5 确定获奖学生逻辑

**获奖学生判定**：
- 从 `student_award_records` 表中查询获奖记录
- 可以根据奖项ID和状态过滤

#### 3.6 设计邮件内容

**邮件模板设计**：
- 奖项名称
- 获奖等级
- 颁奖单位
- 颁奖时间
- 其他相关信息

### 4. 代码修改点

**后端修改**：
1. `Award` 实体类：添加/确认 `endTime` 字段
2. `Student` 实体类：添加/确认 `email` 字段
3. `application.yml`：添加邮件发送配置
4. `EmailService.java`：创建邮件发送服务
5. `AwardScheduler.java`：创建定时任务类

**数据库修改**：
1. 如果需要，修改 `award` 表添加 `end_time` 字段
2. 如果需要，修改 `student` 表添加 `email` 字段

### 5. 测试要点

1. 邮件发送功能测试
2. 定时任务执行测试
3. 获奖学生判定逻辑测试
4. 邮件内容格式测试

## 三、预期效果

1. 当奖项到达结束时间后，系统自动发送邮件通知获奖学生
2. 邮件内容包含完整的获奖信息
3. 邮件发送记录可追溯（可选，可添加邮件发送记录表）

## 四、实施步骤

1. 检查现有实体类和数据库表结构
2. 添加邮件发送依赖和配置
3. 实现邮件发送服务
4. 实现定时任务
5. 测试功能完整性
6. 部署上线

## 五、注意事项

1. 确保邮件发送配置正确，避免邮件发送失败
2. 定时任务的执行频率要合理，避免过于频繁影响系统性能
3. 考虑并发情况，避免重复发送邮件
4. 添加日志记录，便于调试和追踪
5. 考虑邮件发送失败的重试机制

要实现每次运行后端都清空表内容、重新初始化表结构（包括字段更新）和数据，需要修改两个地方：

**1. 修改 application.yml 配置**
将 `spring.sql.init.mode` 从 `never` 改为 `always`，这样 Spring Boot 会在每次应用启动时执行初始化脚本。

**2. 修改 database\_init.sql 脚本**
当前脚本使用 `CREATE TABLE IF NOT EXISTS`，这种方式不会更新已有表的字段。需要改为：

* 先禁用外键约束

* 按正确顺序删除所有表（避免外键依赖错误）

* 重新创建所有表（包含最新字段）

* 插入初始数据

* 重新启用外键约束

**修改步骤：**

1. 打开 `application.yml` 文件，修改 `spring.sql.init.mode` 为 `always`
2. 修改 `database_init.sql` 脚本：

   * 在创建表之前添加 DROP TABLE 语句（按依赖关系从下往上删除）

   * 保留原有的数据插入逻辑

这样，每次运行后端时，都会：

* 删除所有现有表

* 创建包含最新字段的新表

* 插入初始数据

* 确保数据库结构始终与脚本保持一致

**注意事项：**

* DROP TABLE 语句需要按照外键依赖关系的逆序执行

* 使用 `IF EXISTS` 避免删除不存在的表时出错

* 确保在修改脚本前备份好数据（如果有需要保留的数据）

## 问题分析
在线人数显示为0，即使有两个人在线，可能的原因包括：
1. WebSocket连接未成功建立
2. 用户ID未正确传递或解析
3. sessions Map未正确更新
4. 在线人数获取逻辑有问题

## 修复方案

### 1. 后端添加调试日志
在`ChatWebSocketHandler.java`中添加更多调试日志，特别是在WebSocket连接过程中：

```java
// 在afterConnectionEstablished()方法中添加

// 在getUserIdFromSession()和getTokenFromSession()方法中添加
System.out.println("从URL获取到的queryString: " + queryString);

// 在getOnlineUserIds()方法中添加
System.out.println("当前sessions大小: " + sessions.size());
System.out.println("当前sessions内容: " + sessions.keySet());
```

### 2. 修改在线人数获取逻辑
为了确保获取到正确的在线人数，我们可以添加一个新的端点来直接获取在线人数，而不仅仅是在线用户ID列表：

```java
// 在ChatController.java中添加
@PreAuthorize("hasRole('STUDENT') or hasRole('TEACHER') or hasRole('ADMIN')")
@GetMapping("/online-count")
public ResponseEntity<Map<String, Integer>> getOnlineCount() {
    try {
        int count = chatWebSocketHandler.getOnlineUserCount();
        Map<String, Integer> response = new HashMap<>();
        response.put("onlineCount", count);
        return new ResponseEntity<>(response, HttpStatus.OK);
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

### 3. 前端修改获取在线人数的方法
修改前端`getOnlineUserCount()`方法，同时调用新的端点获取在线人数，以便验证：

```javascript
const getOnlineUserCount = async () => {
  console.log('开始获取在线人数...')
  try {
    // 同时调用两个端点进行验证
    const [userIdsResponse, countResponse] = await Promise.all([
      axios.get('/api/chats/online-users'),
      axios.get('/api/chats/online-count')
    ]);
    
    console.log('获取在线用户ID成功，响应数据:', userIdsResponse.data);
    console.log('获取在线人数成功，响应数据:', countResponse.data);
    
    // 使用userIdsResponse.data.length获取在线人数
    onlineUserCount.value = userIdsResponse.data.length || 0;
    console.log('更新在线人数为:', onlineUserCount.value);
  } catch (error) {
    console.error('获取在线人数失败:', error);
    console.error('错误详情:', error.response?.status, error.response?.data);
  }
}
```

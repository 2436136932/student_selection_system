# 修复评选流程进度条显示问题

## 问题分析
通过分析AwardView.vue文件，发现进度条不显示的原因是`calculateProgress`函数中各阶段完成的判断条件过于严格：
- 教师审批阶段：要求`teacherApprovedCount === applicationCount`（所有申请都必须被教师审批）
- 管理员审批阶段：要求`adminApprovedCount === teacherApprovedCount`（所有教师通过的申请都必须被管理员审批）
- 结果公示阶段：要求`finalApprovedCount === adminApprovedCount`（所有管理员通过的申请都必须被最终公示）

当教师和管理员都审批通过了部分申请，但未达到100%审批率时，进度条会显示为0%或25%，无法真实反映实际进度。

## 解决方案
修改`calculateProgress`函数，将严格的阶段完成判断改为基于实际完成比例的计算方式，每个阶段根据实际完成情况贡献0-25%的进度。

## 具体修改
1. **修改calculateProgress函数**（第1254-1295行）：
   - 移除基于阶段完全完成的判断逻辑
   - 改为基于实际完成比例计算每个阶段的贡献度
   - 每个阶段根据实际完成情况贡献0-25%的进度
   - 累加各阶段贡献度得到整体进度

2. **更新阶段进度计算逻辑**：
   - 学生申请阶段：根据是否有申请计算（0%或25%）
   - 教师审批阶段：根据教师审批通过率计算（0-25%）
   - 管理员审批阶段：根据管理员审批通过率计算（0-25%）
   - 结果公示阶段：根据结果公示率计算（0-25%）

## 预期效果
- 当教师和管理员都审批通过了部分申请时，进度条会显示相应的进度
- 进度条能真实反映评选流程的实际进展
- 各阶段进度计算更符合实际业务场景
- 提升用户体验，让用户清晰了解当前评选进度

## 代码修改示例
```javascript
const calculateProgress = (process) => {
  // 根据当前阶段和实际统计数据计算整体进度
  const { applicationCount, teacherApprovedCount, adminApprovedCount, finalApprovedCount } = process;
  
  // 确保所有统计数据都有值
  const totalApplications = applicationCount || 0;
  const teacherApproved = teacherApprovedCount || 0;
  const adminApproved = adminApprovedCount || 0;
  const finalApproved = finalApprovedCount || 0;
  
  // 如果没有申请，进度为0
  if (totalApplications === 0) {
    return 0;
  }
  
  // 计算各阶段的进度贡献（每个阶段最高贡献25%）
  const studentApplicationProgress = totalApplications > 0 ? 25 : 0;
  const teacherApprovalProgress = (teacherApproved / totalApplications) * 25;
  const adminApprovalProgress = (adminApproved / Math.max(teacherApproved, 1)) * 25;
  const resultAnnouncementProgress = (finalApproved / Math.max(adminApproved, 1)) * 25;
  
  // 计算整体进度
  const totalProgress = Math.round(
    studentApplicationProgress + 
    teacherApprovalProgress + 
    adminApprovalProgress + 
    resultAnnouncementProgress
  );
  
  return Math.min(totalProgress, 100);
}
```

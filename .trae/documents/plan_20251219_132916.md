# 学生成绩权限控制修复计划

## 一、问题分析

### 核心问题
学生登录后能看到所有学生的成绩，而不是只能看到自己的成绩。

### 问题原因
通过分析代码，发现问题出在 `getCurrentStudentNumber()` 方法：

1. **用户名与学号混淆**：方法假设学生的用户名就是学号，但实际情况是用户名可能是姓名（如"赵同学"），而学号是单独的字段（如"S001"）
2. **缺少用户与学生的关联查询**：当前代码没有通过用户名查询到对应的学生学号
3. **角色判断可能不准确**：学生角色的判断逻辑可能存在问题

### 验证方式
从前端截图可以看到：
- 登录用户：赵同学（学生角色）
- 能看到的学生：孙同学、赵同学、钱同学
- 这说明数据过滤没有生效，学生能看到所有成绩

## 二、解决方案

### 核心思路
1. **正确关联用户与学生信息**：通过用户名查询到对应的学生学号
2. **修复角色判断逻辑**：确保正确识别学生角色
3. **确保数据过滤生效**：确保学生只能看到自己的成绩

### 具体实现方案

**添加 `findByUsername` 方法**：

#### 4. 修改 ScoreController 类

**修复 `getCurrentStudentNumber` 方法**：
```java
/**
 * 获取当前学生的学号（仅适用于学生角色）
 * @return 当前学生的学号，如果不是学生则返回null
 */
private String getCurrentStudentNumber() {
    Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
    if (authentication != null) {
        // 检查是否为学生角色
        boolean isStudent = authentication.getAuthorities().stream()
            .anyMatch(a -> a.getAuthority().equals("ROLE_STUDENT"));
        if (isStudent) {
            try {
                String username = authentication.getName();
                // 通过用户名查询用户信息
                User user = userService.findByUsername(username);
                if (user != null) {
                    // 通过用户ID查询学生信息
                    Student student = studentService.findStudentByUserId(user.getId());
                    if (student != null) {
                        return student.getStudentNumber();
                    }
                }
            } catch (Exception e) {
                logger.error("获取当前学生学号失败: {}", e.getMessage());
            }
        }
    }
    return null;
}
```

## 三、预期效果

1. **学生登录后**：只能看到自己的成绩
2. **数据过滤生效**：
   - 分页查询时自动过滤为自己的成绩
   - 课程成绩查询时自动过滤为自己的成绩
   - 成绩统计时只显示自己的统计数据
3. **日志记录**：便于调试和审计

## 四、验证方式

1. 使用学生账号登录系统
2. 访问成绩管理页面
3. 检查是否只能看到自己的成绩
4. 查看后端日志，确认学生学号获取正确
5. 使用不同角色测试，确认权限控制正确

## 五、代码优化建议

1. **统一身份验证**：考虑使用 AOP 统一处理身份验证
2. **缓存学生信息**：避免重复查询数据库
3. **完善日志记录**：添加更详细的日志，便于调试
4. **前端配合**：前端也添加角色判断，隐藏不必要的操作

通过以上修改，可以确保学生只能看到自己的成绩，保护学生隐私，增强系统安全性。

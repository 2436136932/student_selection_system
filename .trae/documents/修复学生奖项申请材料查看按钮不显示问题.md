# 修复学生奖项申请材料查看按钮不显示问题

## 问题分析
从用户提供的截图来看，学生奖项申请页面的操作列只有"取消申请"按钮，没有显示"预览材料"和"下载材料"按钮。这说明我们之前的修改可能存在问题，或者前端代码中存在其他导致按钮不显示的原因。

## 可能的原因
1. **数据问题**：后端可能没有返回`materialPath`字段，或者该字段为null，导致按钮的条件判断不满足
2. **角色判断问题**：虽然修改了`hasRole`函数，但可能还有其他地方的角色判断逻辑存在问题
3. **前端语法问题**：可能存在一些运行时错误，导致按钮不显示
4. **权限问题**：可能存在权限控制问题，导致前端无法获取到完整的数据

## 修复计划

### 1. 检查前端代码

#### 1.1 检查按钮显示条件
- 检查`v-if="scope.row.materialPath"`条件判断是否正确
- 确保`materialPath`字段在数据中存在且不为null

#### 1.2 检查控制台错误
- 使用浏览器开发者工具检查控制台是否有运行时错误
- 查看网络请求，检查后端返回的数据结构

#### 1.3 检查角色判断逻辑
- 检查`hasRole`函数是否正确判断用户角色
- 确保`isStudent`计算属性正确计算

### 2. 检查后端代码

#### 2.1 检查数据库查询
- 确保所有查询语句都包含材料相关字段
- 检查`StudentAwardApplicationMapper.xml`中的所有select语句

#### 2.2 检查实体类
- 确保`StudentAwardApplication`实体类包含材料相关字段
- 检查getter和setter方法是否正确

#### 2.3 检查权限控制
- 确保前端可以获取到完整的数据
- 检查`findApplicationById`方法的权限控制

### 3. 测试修复

#### 3.1 本地测试
- 在本地启动前后端服务
- 使用浏览器访问页面，检查按钮是否显示
- 检查控制台是否有错误

#### 3.2 功能测试
- 测试学生上传材料后，按钮是否显示
- 测试教师和管理员查看申请时，按钮是否显示
- 测试材料预览和下载功能是否正常

## 预期效果
1. 学生申请奖项后，在申请列表中可以看到"预览材料"和"下载材料"按钮
2. 教师和管理员在查看申请时，也可以看到"预览材料"和"下载材料"按钮
3. 点击"预览材料"按钮可以在线预览材料
4. 点击"下载材料"按钮可以下载材料

## 修复步骤
1. 首先检查前端控制台的错误信息和网络请求
2. 然后检查后端返回的数据结构
3. 根据检查结果，修复相应的问题
4. 测试修复效果
5. 部署修复后的代码

通过以上步骤，我们可以定位并修复学生奖项申请材料查看按钮不显示的问题，确保所有角色的用户都可以方便地查看和下载申请材料。

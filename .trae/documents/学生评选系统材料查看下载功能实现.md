# 学生评选系统材料查看下载功能实现

## 需求分析
1. 学生申请奖项上传申请材料后可以查看上传的材料
2. 对应的奖项审批教师可以查看并下载该学生材料
3. 管理员可以查看并下载学生材料

## 当前实现情况

### 后端实现
- ✅ 已实现文件上传功能 (`StudentAwardApplicationController.uploadMaterial`)
- ✅ 已实现文件下载功能 (`StudentAwardApplicationController.downloadMaterial`)
- ✅ `StudentAwardApplication` 实体类已包含材料相关字段
- ✅ 已实现权限控制，确保只有授权用户可以访问材料

### 前端实现
- ✅ 已实现学生申请界面，包含文件上传功能
- ✅ 已实现"查看材料"按钮，用于下载材料
- ✅ 已实现教师和管理员界面，包含材料下载功能

## 实现计划

### 1. 增强后端功能

#### 1.1 添加材料预览接口
- 添加 `previewMaterial` 接口，支持直接访问材料文件
- 实现基于 URL 的材料访问，支持浏览器预览
- 确保预览接口的权限控制与下载接口一致

#### 1.2 优化文件存储结构
- 确保文件存储目录安全，防止未授权访问
- 实现文件路径映射，隐藏真实文件路径

### 2. 增强前端功能

#### 2.1 学生申请界面优化
- 添加材料上传成功后的预览功能
- 在申请列表中显示材料状态和格式
- 优化材料下载按钮的显示逻辑

#### 2.2 教师审批界面优化
- 在审批列表中显示材料信息
- 添加材料预览功能，支持在线查看
- 优化审批流程，支持边查看材料边审批

#### 2.3 管理员审批界面优化
- 与教师审批界面类似，添加材料预览功能
- 支持批量查看和下载材料

#### 2.4 材料预览组件
- 实现支持多种文件格式的预览组件
- 支持图片、PDF、文档等格式的在线预览
- 添加文件格式图标显示

### 3. 优化用户体验

#### 3.1 材料上传优化
- 添加材料上传进度提示
- 支持拖拽上传
- 添加文件格式和大小验证

#### 3.2 材料下载优化
- 添加下载进度提示
- 支持批量下载
- 优化下载链接的生成

#### 3.3 界面交互优化
- 添加材料状态指示器
- 优化按钮样式和位置
- 添加操作反馈提示

## 技术实现细节

### 后端技术栈
- Spring Boot 2.x
- Spring Security
- MyBatis Plus
- 文件存储：本地文件系统

### 前端技术栈
- Vue 3
- Element Plus
- Axios
- 预览组件：PDF.js、docx-preview 等

### 数据库设计
- 利用现有 `student_award_applications` 表的材料相关字段
- 无需新增表结构

3. **兼容性风险**：使用成熟的预览组件，确保跨浏览器兼容性
4. **存储容量风险**：实现文件大小限制和定期清理机制

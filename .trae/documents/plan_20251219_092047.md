# 个人中心头像不显示问题 - 进一步解决方案

## 问题分析

经过上次修改，头像问题仍然存在，可能的原因：

1. **userInfo.avatar初始值问题**：当前初始化为空字符串 `''`，这可能导致组件渲染异常
2. **组件边界情况处理**：Element Plus Avatar 组件对 `src` 属性的空值处理可能存在特殊逻辑
3. **异步数据加载时机**：组件可能在 `getUserInfo` 异步获取数据之前就已经渲染

## 解决方案

### 核心思路

1. **优化初始值设置**：将 `userInfo.avatar` 初始值改为 `null`，更符合 Vue 的响应式最佳实践
2. **简化 src 属性绑定**：直接使用 `:src="userInfo.avatar"`，让组件自己处理空值情况
3. **优化默认图标显示条件**：简化 `v-if` 条件，利用 Vue 的真值判断
4. **添加图片加载错误处理**：为 `el-avatar` 添加 `error` 事件处理，确保在图片加载失败时显示默认图标

### 具体修改步骤

1. **修改 userInfo 初始值**：将 `avatar` 初始值从 `''` 改为 `null`
2. **简化 el-avatar 配置**：
   - `:src="userInfo.avatar"`（移除 `|| undefined`）
   - `v-if="!userInfo.avatar"`（简化条件判断）
3. **添加 error 事件处理**：确保在图片加载失败时显示默认图标

### 修改代码示例

```vue
// 用户信息初始值
const userInfo = ref({
  // ... 其他属性
  avatar: null  // 改为 null，而不是空字符串
})

// el-avatar 组件
<el-avatar
  :size="120"
  :src="userInfo.avatar"
  class="user-avatar"
  @click="showAvatarUpload = true"
  @error="handleAvatarError"
>
  <el-icon v-if="!userInfo.avatar"><UserFilled /></el-icon>
</el-avatar>

// 添加错误处理函数
const handleAvatarError = (error) => {
  // 当图片加载失败时，将 userInfo.avatar 设置为 null，触发默认图标显示
  userInfo.value.avatar = null
}
```

## 预期效果

1. **未登录/初始状态**：显示默认用户图标
2. **用户无头像**：显示默认用户图标
3. **用户有头像**：显示用户上传的头像
4. **头像加载失败**：自动切换到默认用户图标
5. **异步数据加载**：无论数据加载顺序如何，头像都能正确显示

## 技术优势

1. **更符合 Vue 最佳实践**：使用 `null` 作为空值，而不是空字符串
2. **更可靠的组件行为**：利用组件内置的空值处理逻辑
3. **更强的错误处理**：添加 `error` 事件处理，确保在各种情况下都能正确显示头像
4. **更简洁的代码**：简化了条件判断和属性绑定
5. **更好的用户体验**：确保头像始终能正确显示，不会出现空白或错误

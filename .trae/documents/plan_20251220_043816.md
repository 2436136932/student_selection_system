## 实现聊天界面填满页面的修复计划

### 问题分析
聊天界面无法填满页面的原因是App.vue中的全局样式限制：
1. `.main-content` 有 `padding: 25px`
2. `.content-wrapper` 有 `padding: 25px` 和 `border-radius: 12px`
3. 这些样式限制了聊天界面的可用空间

### 修复方案

#### 1. 为聊天页面添加特殊标识
- 在ChatView.vue的根元素添加特殊类名
- 用于针对性地覆盖全局样式

#### 2. 修改全局样式
- 在App.vue中添加针对聊天页面的样式
- 移除聊天页面的padding和border-radius限制

#### 3. 调整聊天界面样式
- 确保ChatView.vue中的元素能完全填满可用空间
- 移除不必要的样式限制

### 具体修改点

#### 前端修改
1. **文件**：`frontend/src/views/ChatView.vue`
   - 在根元素添加特殊类名
   - 确保内部元素能完全填满空间

2. **文件**：`frontend/src/App.vue`
   - 添加针对聊天页面的样式覆盖
   - 移除聊天页面的padding和border-radius

### 预期效果

#### 功能效果
- ✅ 聊天界面完全填满页面
- ✅ 移除不必要的内边距和圆角
- ✅ 保持其他页面的原有样式
- ✅ 聊天功能正常工作

#### 视觉效果
- ✅ 聊天界面占据整个内容区域
- ✅ 无多余空白空间
- ✅ 与整体设计风格协调

### 技术实现

1. **CSS 优先级**：使用更具体的选择器覆盖全局样式
2. **条件样式**：为特定页面添加特殊样式
3. **响应式设计**：确保在不同屏幕尺寸下正常显示

### 实现代码概览

#### ChatView.vue 修改
```vue
<template>
  <div class="chat-container full-page-chat">
    <!-- 聊天界面内容 -->
  </div>
</template>
```

#### App.vue 修改
```css
/* 聊天页面特殊样式 */
.main-content .content-wrapper:has(.full-page-chat) {
  padding: 0;
  border-radius: 0;
  background-color: transparent;
  box-shadow: none;
}

.main-content:has(.full-page-chat) {
  padding: 0;
}
```

# 修复WebSocket连接失败的计划

## 问题分析
从前端日志可以看到，WebSocket连接失败的状态码是1006，表示连接被异常关闭。具体错误是：
```
WebSocket connection to 'ws://localhost:8080/ws/chat?userId=1' failed:
```

这表明浏览器无法与后端建立WebSocket连接。虽然后端代码看起来配置正确，但可能存在以下问题：

1. **WebSocket依赖没有被正确加载**：虽然在pom.xml中添加了依赖，但可能没有被正确加载
2. **WebSocket配置没有被正确注册**：WebSocket处理器可能没有被正确注册到Spring容器中
3. **CORS配置问题**：浏览器可能因为CORS政策阻止WebSocket连接
4. **Spring Security配置问题**：Spring Security可能阻止了WebSocket连接

## 解决方案
1. **检查CORS配置**：确保后端允许WebSocket的跨域请求
2. **修改WebSocket配置**：简化WebSocket配置，确保WebSocket处理器被正确注册
3. **检查Spring Security配置**：确保Spring Security允许WebSocket连接
4. **优化前端代码**：使用更可靠的方式处理WebSocket连接

## 实现步骤

### 1. 检查CORS配置
- 查看`CorsConfig.java`文件，确保允许WebSocket的跨域请求
- 添加WebSocket路径到CORS允许列表

### 2. 修改WebSocket配置
- 简化`WebSocketConfig.java`，确保WebSocket处理器被正确注册
- 移除冗余的配置，确保配置简洁清晰

### 3. 检查Spring Security配置
- 查看`SecurityConfig.java`文件，确保Spring Security允许WebSocket连接
- 添加WebSocket路径到Spring Security的允许列表

### 4. 优化前端WebSocket连接
- 修改`ChatView.vue`中的WebSocket连接代码，添加更可靠的重连机制
- 使用更简单的方式构建WebSocket连接

## 预期效果
- 前端能够成功建立WebSocket连接
- 实时消息推送功能正常工作
- WebSocket连接稳定，不会频繁断开

## 技术说明
- WebSocket连接需要服务器端和客户端的配合
- 服务器端需要正确配置WebSocket处理器和CORS政策
- 客户端需要正确处理连接状态和错误

## 后续建议
- 在开发过程中，建议使用集成开发环境（IDE）来管理依赖和配置
- 定期检查日志，及时发现和解决问题
- 考虑使用成熟的WebSocket库来简化开发

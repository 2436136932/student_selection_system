### 问题分析

学生管理中设置生日为2004-01-01，确定后显示为2023-12-31，日期偏移一天。

**根本原因**：
1. 前端date-picker选择的日期是本地时间（如2004-01-01T00:00:00+08:00）
2. 在handleSubmit函数中，使用`JSON.parse(JSON.stringify(form))`会将Date对象转换为UTC格式的ISO字符串（如"2003-12-31T16:00:00.000Z"）
3. 虽然我们调用了formatDate函数，但此时birthDate已经是UTC字符串，格式化无效
4. 后端接收到UTC字符串后，解析时出现时区转换错误

### 解决方案

修改handleSubmit函数，确保在将Date对象转换为JSON之前，先将其格式化为yyyy-MM-dd字符串。

### 代码修改

**文件**：`frontend/src/views/StudentView.vue`

**位置**：handleSubmit函数（第267-322行）

**修改内容**：
- 移除`JSON.parse(JSON.stringify(form))`的使用
- 使用对象展开运算符创建表单副本
- 在表单副本中直接格式化birthDate（此时仍是Date对象）
- 确保发送给后端的是yyyy-MM-dd格式的字符串

### 修复后代码

```javascript
// 新增/编辑学生
const handleSubmit = () => {
  // 创建表单副本，保留Date对象
  const submitData = { ...form };
  
  // 直接格式化birthDate（此时仍是Date对象）
  submitData.birthDate = formatDate(submitData.birthDate);
  
  if (form.id) {
    // 编辑学生
    axios.put(`/api/students/${form.id}`, submitData)
      .then(response => {
        ElMessage.success('编辑学生成功')
        dialogVisible.value = false
        getStudents()
      })
      .catch(error => {
        ElMessage.error('编辑学生失败')
        console.error('Error updating student:', error)
      })
  } else {
    // 新增学生
        console.log('提交的表单数据:', submitData)
        axios.post('/api/students', submitData)
          .then(response => {
            console.log('新增学生响应:', response)
            ElMessage.success('新增学生成功')
            dialogVisible.value = false
            getStudents()
          })
          .catch(error => {
            ElMessage.error('新增学生失败')
            console.error('Error creating student:', error)
            if (error.response) {
              console.error('错误响应数据:', error.response.data)
              console.error('错误响应状态:', error.response.status)
              console.error('错误响应头:', error.response.headers)
            } else if (error.request) {
              console.error('请求发送失败:', error.request)
            } else {
              console.error('请求配置错误:', error.message)
            }
          })
  }
}
```

### 预期效果

1. 新增学生时，选择2004-01-01，提交后显示正确的2004-01-01
2. 编辑学生时，出生日期显示正确，保存后不会偏移
3. 前后端日期处理保持一致，不再出现时区转换错误

# 修复教师审批统计数据不显示的问题

## 问题分析
从截图和代码分析来看，教师审批统计数据不显示的原因是：
- `countTeacherApprovedApplicationsByAwardId`方法使用`status = 1`来统计教师审核通过的数量
- 但当管理员审批通过后，申请的状态会变为3（管理员审批通过）
- 这导致教师审批通过的统计方法无法统计到已经被管理员审批的申请
- 同样的问题也可能影响其他统计方法

## 解决方案
修改统计方法，使用专门的审批状态字段来统计各阶段的审批数量，而不是使用整体的`status`字段：

1. **修改`countTeacherApprovedApplicationsByAwardId`方法**：
   - 将查询条件从`status = 1`改为`teacherApprovalStatus = 1`
   - 这样无论最终状态是什么，都能正确统计教师审批通过的数量

2. **修改`countAdminApprovedApplicationsByAwardId`方法**：
   - 将查询条件从`status = 3`改为`adminApprovalStatus = 1`
   - 确保能正确统计管理员审批通过的数量

3. **检查并修改其他相关统计方法**：
   - 确保所有统计方法都使用正确的字段来统计各阶段的审批数量

## 具体修改
1. **修改`StudentAwardApplicationServiceImpl.java`文件**：
   - 第510-514行：`countTeacherApprovedApplicationsByAwardId`方法
   - 第519-523行：`countAdminApprovedApplicationsByAwardId`方法

2. **更新查询条件**：
   - 教师审批统计：使用`teacherApprovalStatus = 1`
   - 管理员审批统计：使用`adminApprovalStatus = 1`

## 预期效果
- 教师审批通过的统计数据能正确显示，无论申请是否已经被管理员审批
- 管理员审批通过的统计数据能正确显示
- 流程进度能根据实际审批情况正确计算
- 统计数据与实际审批情况保持一致

## 代码修改示例
```java
// 修改前
public long countTeacherApprovedApplicationsByAwardId(Long awardId) {
    return studentAwardApplicationMapper.selectCount(
            new LambdaQueryWrapper<StudentAwardApplication>()
                    .eq(StudentAwardApplication::getAwardId, awardId)
                    .eq(StudentAwardApplication::getStatus, 1) // 1表示教师审核通过
    );
}

// 修改后
public long countTeacherApprovedApplicationsByAwardId(Long awardId) {
    return studentAwardApplicationMapper.selectCount(
            new LambdaQueryWrapper<StudentAwardApplication>()
                    .eq(StudentAwardApplication::getAwardId, awardId)
                    .eq(StudentAwardApplication::getTeacherApprovalStatus, 1) // 1表示教师审核通过
    );
}
```

# 实现左侧用户列表显示未读消息来源

## 问题分析
目前系统已经实现了在聊天中心图标上显示未读消息总数，但还需要在左侧用户列表中显示每个用户的未读消息数量，以便用户知道是谁发送了未读消息。

当前实现的问题：
- `getUnreadCount`方法依赖于`chatSessions`中的`unreadCount`字段
- 后端返回的会话数据中可能没有包含`unreadCount`字段
- 用户列表中无法显示每个用户的未读消息数量

## 解决方案
1. **后端修改**：
   - 修改`ChatController`中的`getCurrentUserChatSessions`方法，为每个会话添加未读消息数量字段
   - 在`ChatMessageService`中添加获取会话未读消息数量的方法
   - 在返回的会话数据中包含未读消息数量

2. **前端修改**：
   - 修改`ChatList.vue`组件，确保`getUnreadCount`方法能够正确获取每个用户的未读消息数量
   - 更新UI显示，在用户列表中清晰显示每个用户的未读消息数量
   - 确保未读消息数量能够实时更新

## 具体修改

### 后端修改
1. **修改`ChatMessageService`接口**：
   - 添加获取会话未读消息数量的方法

2. **修改`ChatController`**：
   - 在`getCurrentUserChatSessions`方法中，为每个会话添加`unreadCount`字段
   - 调用`ChatMessageService`的方法获取未读消息数量

3. **修改`ChatSessionService`**：
   - 添加相关方法支持未读消息数量的获取

### 前端修改
1. **修改`ChatList.vue`**：
   - 确保`getUnreadCount`方法能够正确获取每个用户的未读消息数量
   - 更新用户列表项，显示未读消息数量

2. **修改`ChatView.vue`**：
   - 确保`fetchChatSessions`方法能够正确处理包含未读消息数量的会话数据
   - 当收到新消息时，更新对应会话的未读消息数量

3. **修改CSS样式**：
   - 确保未读消息数量在用户列表中显示清晰
   - 添加适当的样式，提高用户体验

## 预期效果
- 左侧用户列表中每个用户项会显示对应的未读消息数量
- 用户可以清晰地知道是谁发送了未读消息
- 未读消息数量会实时更新
- 进入聊天窗口后，对应用户的未读消息数量会清零

## 实现步骤
1. 后端实现未读消息数量的计算和返回
2. 前端更新会话数据处理逻辑
3. 前端更新用户列表UI，显示未读消息数量
4. 测试未读消息数量的实时更新
5. 测试进入聊天窗口后未读消息数量的清零

### 前端示例
```javascript
// 在ChatList.vue中更新getUnreadCount方法
const getUnreadCount = (user) => {
  // 从chatSessions中查找与当前用户相关的会话
  const session = props.chatSessions.find(s => 
    s.user1Id === user.id || s.user2Id === user.id
  );
  // 返回会话的未读消息数量，如果没有会话或没有未读消息，返回0
  return session?.unreadCount || 0;
};
```

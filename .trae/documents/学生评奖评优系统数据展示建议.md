# 学生评奖评优系统数据可视化计划

## 一、数据选择与图表设计

### 1. 奖项级别分布
- **数据来源**：`awards` 表的 `award_level` 字段
- **图表类型**：饼图
- **功能描述**：直观展示不同级别奖项的数量占比，帮助用户了解奖项设置结构
- **展示内容**：国家、省级、校级、院级奖项的数量和占比

### 2. 奖项申请状态分布
- **数据来源**：`student_award_applications` 表的 `status` 字段
- **图表类型**：环形图
- **功能描述**：展示不同审批状态的申请数量分布，反映评选流程的整体进展
- **展示内容**：待审核、教师通过、教师拒绝、管理员通过、管理员拒绝的申请数量和占比

### 3. 学生获奖情况（按专业）
- **数据来源**：`student_award_records` 表的 `major` 和 `student_id` 字段
- **图表类型**：柱状图
- **功能描述**：对比不同专业的学生获奖数量，了解各专业的获奖情况
- **展示内容**：各专业的获奖学生总数，按专业名称排序

### 4. 审批流程时间分析
- **数据来源**：`student_award_applications` 表的 `application_time`、`teacher_approval_time`、`admin_approval_time` 字段
- **图表类型**：瀑布图
- **功能描述**：展示从申请到最终审批的各阶段平均耗时，帮助优化审批流程
- **展示内容**：申请到教师审批、教师审批到管理员审批、管理员审批到最终结果的平均耗时

### 5. 奖项申请趋势
- **数据来源**：`student_award_applications` 表的 `application_time` 字段
- **图表类型**：折线图
- **功能描述**：展示奖项申请的时间趋势，了解学生的申请行为模式
- **展示内容**：按时间维度（日/周/月）统计的申请数量变化

## 二、技术实现方案

### 1. 前端技术栈
- **图表库**：ECharts（功能强大，支持多种图表类型）
- **框架**：Vue 3（与现有前端框架保持一致）
- **组件化设计**：将每个图表封装为独立组件，便于复用和维护

### 2. 后端 API 设计
- **接口1**：`GET /api/statistics/award-level-distribution` - 获取奖项级别分布数据
- **接口2**：`GET /api/statistics/application-status-distribution` - 获取申请状态分布数据
- **接口3**：`GET /api/statistics/awards-by-major` - 获取各专业获奖情况数据
- **接口4**：`GET /api/statistics/approval-time-analysis` - 获取审批流程时间分析数据
- **接口5**：`GET /api/statistics/application-trend` - 获取奖项申请趋势数据

### 3. 数据处理流程
1. 前端组件挂载时调用对应 API 接口
2. 后端从数据库查询原始数据
3. 后端对数据进行统计分析和格式化
4. 返回 JSON 格式的数据给前端
5. 前端使用 ECharts 渲染图表

## 三、页面布局设计

### 1. 数据统计页面
- **位置**：新增独立的数据统计页面，或在首页添加数据可视化区域
- **布局**：采用响应式网格布局，在不同屏幕尺寸下自动调整
- **组件排列**：
  - 顶部：系统概览卡片（学生总数、奖项总数、申请总数、已通过申请数）
  - 中部：图表区域，采用 2x2 网格布局（奖项级别分布、申请状态分布、学生获奖情况、审批流程时间分析）
  - 底部：奖项申请趋势图（占满整行）

### 2. 权限控制
- 学生端：仅能查看个人相关的统计数据
- 教师端：能查看所带学生的统计数据和审批相关数据
- 管理员端：能查看全系统的所有统计数据

## 四、实现步骤

### 1. 后端实现（1-2天）
- 设计并实现 5 个统计 API 接口
- 编写数据查询和统计逻辑
- 测试 API 接口的正确性和性能

### 2. 前端实现（2-3天）
- 集成 ECharts 库到前端项目
- 封装 5 个图表组件
- 实现数据统计页面布局
- 集成 API 接口，实现数据动态加载

### 3. 测试与优化（1天）
- 测试图表在不同浏览器中的兼容性
- 优化图表加载性能
- 调整图表样式，确保视觉一致性
- 测试不同角色的权限控制

### 4. 部署上线（0.5天）
- 将代码合并到主分支
- 部署到测试环境进行验收
- 部署到生产环境

## 五、预期效果

1. **直观的数据展示**：通过图表形式直观展示系统核心数据，替代传统的表格展示
2. **实时数据更新**：图表数据实时从数据库获取，确保数据的准确性和时效性
3. **多维度数据分析**：从不同角度分析数据，帮助用户更好地理解系统运行情况
4. **响应式设计**：在不同设备上都能良好展示
5. **良好的用户体验**：图表支持交互（如悬停显示详情、点击查看更多）

## 六、扩展建议

1. 后续可根据用户反馈，增加更多图表类型和数据维度
2. 支持图表数据导出功能，方便用户进行离线分析
3. 实现自定义报表功能，允许用户根据需求组合不同图表
4. 增加数据预警功能，当某些指标超出阈值时自动提醒管理员

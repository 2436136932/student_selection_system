# 修复发送验证码网络连接问题

## 问题分析
从错误信息中可以看到：
1. **网络错误**：`POST http://localhost:8080/api/auth/send-reset-code net::ERR_CONNECTION_REFUSED`
2. **Axios错误**：`AxiosError {message: 'Network Error', name: 'AxiosError', code: 'ERR_NETWORK'}`
3. **状态监听错误**：`{newCodeSending: undefined, newCountdown: undefined}`

这表明前端无法连接到后端服务器，可能的原因包括：
- 后端服务器没有运行
- 后端服务器运行在不同的端口
- 网络连接问题
- 后端服务配置错误

## 修复方案
1. **检查后端服务器状态**：确认后端服务器是否正在运行
2. **启动后端服务器**：如果后端服务器没有运行，使用Maven启动它
3. **检查端口配置**：确认后端服务器运行在8080端口
4. **验证API端点**：确保`/api/auth/send-reset-code`端点存在且可用
5. **修复状态监听**：解决状态变化监听中的undefined问题

## 预期效果
- 后端服务器成功启动并运行在8080端口
- 前端能够成功连接到后端服务器
- 发送验证码功能正常工作
- 状态变化监听显示正确的状态值

## 具体实现
1. 检查当前运行的进程，确认后端服务器是否正在运行
2. 使用Maven启动后端Spring Boot应用
3. 等待后端服务器完全启动
4. 测试发送验证码功能
5. 修复状态变化监听中的问题

## 测试方法
1. 刷新页面
2. 打开忘记密码对话框
3. 输入邮箱地址
4. 点击发送验证码按钮
5. 观察控制台日志和页面反馈
6. 确认验证码发送成功并开始倒计时

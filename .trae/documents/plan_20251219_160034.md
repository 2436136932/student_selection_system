# 修复WebSocket连接出错的计划

## 问题分析
从后端日志可以看出，WebSocket连接出错的原因是`TextWebSocketHandler`类找不到。具体错误是：
```
Caused by: java.io.FileNotFoundException: class path resource [org/springframework/web/socket/handler/TextWebSocketHandler.class] cannot be opened because it does not exist
```

这表明虽然我们在pom.xml中添加了`spring-boot-starter-websocket`依赖，但该依赖没有被正确加载或没有包含`TextWebSocketHandler`类。

## 解决方案
1. **停止所有后端服务**：确保所有正在运行的后端服务都被关闭
2. **重新编译项目**：确保新添加的WebSocket依赖被正确加载
3. **重启后端服务**：检查WebSocket服务是否正常运行
4. **优化前端代码**：使用更可靠的方式处理WebSocket连接

## 实现步骤

### 1. 停止所有后端服务
- 停止所有正在运行的`mvn spring-boot:run`命令
- 确保没有任何后端服务占用8080端口

### 2. 重新编译项目
- 执行`mvn clean compile`命令，重新编译项目
- 检查编译过程中是否有任何错误

### 3. 重启后端服务
- 执行`mvn spring-boot:run`命令，启动后端服务
- 检查日志，确保WebSocket服务正常启动

### 4. 优化前端WebSocket连接
- 修改`ChatView.vue`中的WebSocket连接代码，添加更详细的错误处理
- 使用更可靠的方式构建WebSocket连接

## 预期效果
- 后端服务成功启动，WebSocket服务正常运行
- 前端能够成功建立WebSocket连接
- 实时消息推送功能正常工作

## 技术说明
- WebSocket依赖需要被正确加载，才能提供`TextWebSocketHandler`类
- 重新编译和启动项目可以确保依赖被正确加载
- 优化前端代码可以提高WebSocket连接的可靠性

## 后续建议
- 在开发过程中，建议使用集成开发环境（IDE）来管理依赖和启动服务
- 定期更新依赖版本，确保依赖之间的兼容性
- 实现更完善的错误处理，提高系统的鲁棒性

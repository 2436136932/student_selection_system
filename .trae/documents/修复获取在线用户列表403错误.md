## 问题分析
ChatView.vue 中调用 `/api/chats/online-users` 接口返回 403 Forbidden 错误，原因是 JWT 认证过滤器无法正确获取 Authorization 头。

## 根本原因
在 `JwtAuthenticationFilter.java` 中，`@Value("${jwt.header}")` 注解没有被正确处理，导致 `jwtHeader` 变量无法获取到配置值，从而无法从请求头中提取 JWT token，最终导致权限验证失败。

## 修复方案
修改 `JwtAuthenticationFilter.java` 文件，确保 `jwtHeader` 变量能正确获取到 Authorization 头名：

1. 将 `jwtHeader` 变量从依赖 `@Value` 注解改为硬编码为 "Authorization"
2. 确保在 `doFilterInternal` 方法中能正确获取到请求头中的 JWT token

## 修复代码
```java
// 将 JwtAuthenticationFilter.java 中的 jwtHeader 变量定义修改为：
private final String jwtHeader = "Authorization";

// 删除原有的 @Value 注解
```

## 预期效果
修复后，`/api/chats/online-users` 接口将能正确验证 JWT token，返回在线用户列表，解决 403 Forbidden 错误。

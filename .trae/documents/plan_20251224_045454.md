## 问题分析
在线人数统计不准确的根本原因是：
1. App.vue中添加了WebSocket连接初始化逻辑，确保用户一登录就建立连接
2. ChatView.vue中仍保留原有的WebSocket连接初始化逻辑，在进入聊天中心时会再次建立连接
3. 这导致重复连接，可能引起冲突，影响在线人数统计
4. 同时，ChatView.vue中的WebSocket连接包含完整的消息处理逻辑，而App.vue中的只有基本连接逻辑

## 修复方案

### 方案1：统一WebSocket连接管理
将WebSocket连接的管理统一到一个地方，确保每个用户只有一个WebSocket连接：

1. **保留ChatView.vue中的完整WebSocket实现**：
   - 包含消息处理、重连机制等完整功能

2. **修改ChatView.vue，使其支持在后台运行**：
   - 将WebSocket连接对象提升为全局状态或使用事件总线
   - 确保用户进入系统后自动建立连接，离开聊天中心时不关闭连接

3. **移除App.vue中的WebSocket连接逻辑**：
   - 避免重复连接

### 方案2：将WebSocket连接逻辑迁移到单独的服务
创建一个单独的WebSocket服务，统一管理所有WebSocket连接：

1. **创建websocketService.js**：
   - 封装WebSocket连接、消息处理、重连机制
   - 提供全局访问方法

2. **在App.vue中初始化WebSocket连接**：
   - 确保用户一登录就建立连接

3. **在ChatView.vue中使用WebSocket服务**：
   - 监听消息
   - 发送消息
   - 移除重复的连接逻辑

## 推荐方案
方案2更优，因为它提供了更好的模块化和可维护性。具体实现如下：

### 实现步骤

1. **创建websocketService.js**：
   ```javascript
   // src/utils/websocketService.js
   import { ref } from 'vue'
   
   const socket = ref(null)
   const isConnected = ref(false)
   const messageHandler = ref(null)
   
   export const initializeWebSocket = (onMessage) => {
     if (socket.value?.readyState === WebSocket.OPEN) {
       console.log('WebSocket连接已存在')
       if (onMessage) {
         messageHandler.value = onMessage
       }
       return
     }
     
     const userId = JSON.parse(localStorage.getItem('userInfo'))?.id
     if (!userId) return
     
     try {
       const token = localStorage.getItem('token')
       if (!token) {
         console.log('没有有效的token，无法建立WebSocket连接')
         return
       }
       
       socket.value = new WebSocket(`ws://localhost:8080/ws/chat?userId=${userId}&token=${token}`)
       
       socket.value.onopen = () => {
         console.log('WebSocket连接已建立')
         isConnected.value = true
       }
       
       socket.value.onmessage = (event) => {
         console.log('收到WebSocket消息:', event.data)
         if (messageHandler.value) {
           messageHandler.value(event)
         }
       }
       
       socket.value.onclose = (event) => {
         console.log(`WebSocket连接已关闭，代码: ${event.code}, 原因: ${event.reason}`)
         isConnected.value = false
         // 5秒后重新连接
         setTimeout(() => {
           initializeWebSocket(messageHandler.value)
         }, 5000)
       }
       
       socket.value.onerror = (error) => {
         console.error('WebSocket连接出错:', error)
       }
       
       if (onMessage) {
         messageHandler.value = onMessage
       }
     } catch (error) {
       console.error('创建WebSocket连接失败:', error)
       // 5秒后重新连接
       setTimeout(() => {
         initializeWebSocket(messageHandler.value)
       }, 5000)
     }
   }
   
   export const sendMessage = (message) => {
     if (socket.value?.readyState === WebSocket.OPEN) {
       socket.value.send(JSON.stringify(message))
     }
   }
   
   export const getConnectionStatus = () => isConnected.value
   ```

2. **修改App.vue**：
   - 移除原有的WebSocket连接逻辑
   - 导入并使用websocketService初始化连接

3. **修改ChatView.vue**：
   - 移除原有的WebSocket连接初始化逻辑
   - 导入并使用websocketService处理消息
   - 只保留消息处理的业务逻辑

4. **确保WebSocket连接在用户登录时初始化**：
   - 在App.vue的onMounted钩子中调用initializeWebSocket
   - 确保每个用户只有一个WebSocket连接

### 预期效果
- 每个用户在登录系统时建立一个WebSocket连接
- 离开聊天中心后，WebSocket连接不会断开
- 在线人数统计准确，反映真实的登录用户数量
- WebSocket连接管理更加模块化和可维护

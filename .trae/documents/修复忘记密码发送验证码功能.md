# 修复忘记密码发送验证码功能

## 问题分析
点击忘记密码中的发送验证码没有反应，经过分析发现：

1. **根本原因**：`application.yml` 文件中邮件配置的缩进不正确，导致 Spring Boot 无法读取邮件配置
2. **直接原因**：邮件配置未正确嵌套在 `spring` 配置下，Spring 无法创建 `JavaMailSender` Bean
3. **连锁反应**：`EmailServiceImpl` 依赖 `JavaMailSender` 无法创建，进而导致 `VerificationCodeServiceImpl` 和 `AuthController` 无法创建
4. **最终结果**：`/api/auth/send-reset-code` 端点无法访问，点击发送验证码按钮没有反应

## 修复步骤

1. **修复邮件配置缩进**：
   - 修改 `application.yml` 文件，将邮件配置正确嵌套在 `spring` 配置下
   - 确保第49行的 `# Email Configuration` 注释没有多余空格
   - 确保第50-62行的 `mail` 配置正确缩进

2. **重启应用**：
   - 重新启动 Spring Boot 应用，确保所有 Bean 能够正常创建
   - 检查日志，确认没有与邮件服务相关的错误

3. **测试功能**：
   - 打开登录页面，点击"忘记密码？"
   - 输入注册邮箱，点击"发送验证码"
   - 检查是否收到验证码邮件
   - 验证验证码是否能正确用于重置密码

## 预期结果
- 点击"发送验证码"按钮后，按钮显示"发送中..."
- 发送成功后显示"验证码发送成功"提示
- 按钮开始60秒倒计时
- 用户能收到包含验证码的邮件
- 验证码能正确用于重置密码

## 部署步骤

### 1. 安装 Java 21

CentOS 7 yum 源没有 Java 21，从 Oracle 官网下载 RPM 包安装：

```bash
wget -q --no-check-certificate https://download.oracle.com/java/21/latest/jdk-21_linux-x64_bin.rpm -O /tmp/jdk-21.rpm
rpm -ivh /tmp/jdk-21.rpm
```

### 2. 安装 Nginx

CentOS 7 yum 源没有 Nginx，手动添加官方源安装：

```bash
# 添加 nginx 源
cat > /etc/yum.repos.d/nginx.repo << 'EOF'
[nginx-stable]
name=nginx stable repo
baseurl=http://nginx.org/packages/centos/7/x86_64/
gpgcheck=0
enabled=1
EOF

# 安装依赖
yum install -y pcre2

# 安装 nginx
rpm -Uvh http://nginx.org/packages/centos/7/x86_64/RPMS/nginx-1.24.0-1.el7.ngx.x86_64.rpm
```

### 3. 配置 MySQL

```bash
# 启动 MySQL
service mysqld start

# 重置 root 密码（通过 skip-grant-tables 方式）
service mysqld stop
mysqld_safe --skip-grant-tables &
# 连接后执行：
# USE mysql; UPDATE user SET password=PASSWORD('root123') WHERE user='root'; FLUSH PRIVILEGES;
service mysqld restart

# 创建数据库
mysql -u root -proot123 -e "CREATE DATABASE student_selection_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 导入数据（需先将 database_init.sql 上传到服务器）
mysql -u root -proot123 student_selection_system < /tmp/database_init_fixed.sql
```

**注意：** MySQL 5.5 不支持 `DATETIME DEFAULT CURRENT_TIMESTAMP`，需要将 SQL 文件中的该语法替换为 `DATETIME NULL DEFAULT NULL`。使用 Node.js 脚本处理避免编码问题：

```javascript
// fix_sql.js - 修复 SQL 文件兼容 MySQL 5.5
const fs = require('fs');
let content = fs.readFileSync('/tmp/database_init.sql', 'utf8');
content = content.replace(/DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP/g, 'DATETIME NULL DEFAULT NULL');
content = content.replace(/DATETIME DEFAULT CURRENT_TIMESTAMP/g, 'DATETIME NULL DEFAULT NULL');
// 去除末尾注释行（MySQL 5.5 会报错）
content = content.split('\n').filter(line => !line.trim().startsWith('--')).join('\n');
fs.writeFileSync('/tmp/database_init_fixed.sql', content);
```

### 4. 配置 Nginx 反向代理

配置文件：`/etc/nginx/conf.d/student-selection-system.conf`

```nginx
server {
    listen 80;
    server_name _;
    root /www/wwwroot/student-selection-system/frontend;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /uploads {
        alias /www/wwwroot/student-selection-system/uploads;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 1000;
}
```

### 7. 前端 API 路径修复（重要）

前端代码中所有 API 请求必须使用相对路径 `/api`，不能写死 `http://localhost:8080`。

**需要修改的文件（共 10 个）：**
- `src/App.vue`
- `src/components/ChatList.vue`
- `src/components/ChatWindow.vue`
- `src/components/MaterialPreview.vue`
- `src/views/CarouselManage.vue`
- `src/views/ChatView.vue`
- `src/views/HomeView.vue`
- `src/views/ProfileView.vue`
- `src/views/StudentAwardApplicationView.vue`
- `src/utils/websocketService.js`

**修复内容：**
1. 将 `http://localhost:8080/api` 替换为空字符串（保留 `/api` 前缀）
2. 将 WebSocket 连接 `ws://localhost:8080/ws/chat` 替换为 `` `ws://${window.location.host}/ws/chat` ``

**推荐使用 Node.js 脚本批量修复（避免 PowerShell 编码问题）：**

```javascript
// fix_api_paths.js
const fs = require('fs');
const path = require('path');

const files = [
  'src/App.vue', 'src/components/ChatList.vue', 'src/components/ChatWindow.vue',
  'src/components/MaterialPreview.vue', 'src/views/CarouselManage.vue',
  'src/views/ChatView.vue', 'src/views/HomeView.vue', 'src/views/ProfileView.vue',
  'src/views/StudentAwardApplicationView.vue', 'src/utils/websocketService.js'
];

const baseDir = '/frontend';
for (const file of files) {
  const fullPath = path.join(baseDir, file);
  let content = fs.readFileSync(fullPath, 'utf8');
  content = content.replace(/http:\/\/localhost:8080\/api/g, '/api');
  content = content.replace(/http:\/\/localhost:8080/g, '');
  if (file === 'src/utils/websocketService.js') {
    content = content.replace(/ws:\/\/localhost:8080\/ws\/chat/g, 'ws://${window.location.host}/ws/chat');
  }
  fs.writeFileSync(fullPath, content);
  console.log('Fixed:', file);
}
```

**修复后重新构建并部署：**
```bash
cd frontend
npm run build
# 打包上传
zip -r /tmp/frontend-dist.zip dist/*
scp -i temp_key /tmp/frontend-dist.zip root@43.226.36.54:/tmp/

```

## 常见问题排查

### 聊天中心报 ERR_CONNECTION_REFUSED

**原因：** 前端代码中 API 请求写死了 `http://localhost:8080`，线上无法连接。

**修复：** 参考「前端 API 路径修复」章节，将所有 `http://localhost:8080` 替换为空，WebSocket 改为动态获取 host。

### 502 Bad Gateway

**原因：** 后端服务未启动或崩溃。

```bash
systemctl status student-selection-system   # 查看状态
systemctl restart student-selection-system  # 重启
tail -f /www/wwwroot/student-selection-system/logs/app-error.log  # 查看错误日志
```

### 静态资源 404

**原因：** Nginx 配置中 root 路径错误或前端文件未正确部署。

```bash
ls -la /www/wwwroot/student-selection-system/frontend/  # 确认文件存在
nginx -t                                               # 测试配置
```

### 数据库连接失败

**原因：** MySQL 服务停止或密码错误。

```bash
service mysqld status  # 查看 MySQL 状态
service mysqld start   # 启动 MySQL
```

### 端口被占用

## 域名配置（No-IP）

### 1. 在 No-IP 上配置 A 记录

登录 No-IP 控制台（https://www.noip.com/），进入你的域名管理页面：

1. 找到你注册的域名，点击 **"Edit"** 或管理按钮
2. 在 DNS 记录中添加/编辑 **A 记录**：
   - **类型**：A
   - **主机名**：留空（或填 @，表示根域名）
   - **值/目标**：43.226.36.54（你的服务器 IP）
   - **TTL**：3600（默认即可）
3. 保存后等待 DNS 生效（通常 5-30 分钟，最长不超过 48 小时）

**验证 DNS 生效：**
`ash
# 在你的 Windows 电脑上执行
nslookup 你的域名.np-ip.org
# 应该返回 43.226.36.54
`

或者在浏览器访问 http://你的域名.np-ip.org，看是否能打开网站。

### 2. 在 Nginx 中配置 server_name

通过宝塔面板或 SSH 修改 Nginx 配置文件 /etc/nginx/conf.d/student-selection-system.conf：

将原来的 server_name _; 改为你的域名：

`
ginx
server {
    listen 80;
    server_name 你的域名.np-ip.org;    # ← 改成你的 No-IP 域名
    root /www/wwwroot/student-selection-system/frontend;
    index index.html;

    # ... 其他配置保持不变 ...
}
`

**通过宝塔面板操作（推荐）：**
1. 打开 http://43.226.36.54:8888/
2. 进入 **网站** → 找到当前网站 → 点击 **设置**
3. 在 **域名管理** 中添加你的 No-IP 域名
4. 点击 **保存**

**通过 SSH 操作：**
`ash
# 编辑配置文件
vi /etc/nginx/conf.d/student-selection-system.conf
# 修改 server_name 行

# 测试配置
nginx -t

# 重载 Nginx
nginx -s reload
`

### 3. 配置 HTTPS（可选但推荐）

No-IP 免费域名不支持直接申请 SSL 证书。有两种方案：

**方案 A：使用 Cloudflare 中转（推荐，免费 HTTPS）**
1. 注册 Cloudflare 账号（https://www.cloudflare.com/）
2. 将域名的 DNS 托管迁移到 Cloudflare
3. 在 Cloudflare 中开启 **SSL/TLS → Full (Strict)**
4. Cloudflare 会自动为你的网站提供 HTTPS

**方案 B：自签名证书（仅测试用）**
`ash
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/selfsigned.key \
  -out /etc/nginx/ssl/selfsigned.crt
`
然后在 Nginx 配置中添加 443 端口监听。

### 4. 更新前端访问地址

配置域名后，用户通过域名访问，前端代码中的相对路径 /api 无需修改，因为域名访问时浏览器会自动使用当前域名。

如果之前有写死的 IP 地址（如 http://43.226.36.54:8080），需要改回相对路径。
